# Shared Planning And Execution

## When To Use
- New feature work with multiple files or dependencies.
- Refactors where scope or regression risk is non-trivial.
- Tasks requiring explicit milestones and verification.

## Checklist
1. Clarify scope, assumptions, and acceptance criteria.
2. Identify impacted files and dependencies.
3. Split work into small, ordered, verifiable steps.
4. Define verification for each step (lint/test/build/checklist).
5. Call out technical risks and rollback path when relevant.
6. Execute step-by-step and re-check scope after each milestone.

## Output Format
- Scope and assumptions
- Step plan
- Verification per step
- Risks and rollback notes
- Completion status
