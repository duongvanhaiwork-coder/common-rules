# Shared Git Workflow

## When To Use
- When preparing commits, PRs, or release branches.
- When asked to cleanly package and deliver changes.

## Checklist
1. Inspect status and diff before staging.
2. Stage only relevant files for the intended scope.
3. Avoid destructive commands unless explicitly requested.
4. Do not commit or push unless explicitly requested.
5. Use clear commit messages focused on intent and impact.
6. Confirm verification state before merge/PR handoff.

## Output Format
- Included scope
- Commit/PR readiness
- Verification state
- Risks or follow-up items
