# Shared Communication Handoff

## When To Use
- After implementation, review, or debugging tasks.
- Any time work is handed off to another engineer or session.

## Checklist
1. Summarize what changed and why.
2. List touched files or modules.
3. Report verification status and evidence.
4. Call out risks, assumptions, and open questions.
5. Provide concrete next steps for continuation.
6. Explicitly note behavior changes and breaking impact.

## Output Format
- What changed
- Why it changed
- Verification status
- Risks and assumptions
- Next actions
